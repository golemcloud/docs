# Making custom APIs

The [Worker Gateway](/concepts/worker-gateway) service not only exposes the [low-level REST API for invocations](/invoke/http) but also provides a way to define fully customizable HTTP APIs.
Future versions will support other protocols as well such as gRPC or GraphQL.

### API Development Workflow

1. **[Define an API Definition](making-custom-apis/define-api-definition)**: Learn how to create a new API definition.
2. **[Register or Upload the API Definition](making-custom-apis/register-api-definition)**: Register or upload your API definition to the platform.
3. **[Deploy the API Definition](making-custom-apis/deploy-api-definition)**: Deploy your API to make it accessible.
4. **[Try Out the Deployed API](making-custom-apis/try-out-deployed-api)**: Test the functionality of your deployed API.
5. **[Update API Definitions and Deployments](making-custom-apis/update-api-deployment)**: Make changes to existing APIs and redeploy.
6. **[Set Up API Authentication](making-custom-apis/authentication)**: Configure authentication mechanisms for your API.
7. **[Configure CORS for the API Gateway](making-custom-apis/cors)**: Set up Cross-Origin Resource Sharing (CORS) policies.
8. **[Map Responses with Rib](making-custom-apis/rib-in-api-definition)**: Use Rib for customizing API response mappings.
9. **[Automatic Documentation for Endpoints](making-custom-apis/spec-inference)**: Learn more on automatic spec generation based on Rib scripts.
10. **[Constraints on Component Updates](making-custom-apis/component-constraints)**: Learn more on restrictions for updating components after deployment.
11. **[Import OpenAPI Specifications](making-custom-apis/import-openapi-spec)**: Convert OpenAPI specs into Golem API definitions.
12. **[Integrate with External API Gateways](making-custom-apis/external-api-gateways)**: Connect your APIs to existing gateways.
13. **[Troubleshoot Common Issues](making-custom-apis/trouble-shooting)**: Resolve potential problems with your APIs.
