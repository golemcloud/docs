# Deploy to Kubernetes

To get started we recommend using the [Golem Helm Chart](https://github.com/golemcloud/golem/blob/main/kube/golem-chart).
You will need to have [Helm](https://helm.sh/docs/intro/quickstart/) and [kubectl](https://kubernetes.io/docs/tasks/tools/) installed locally
and running [kubernetes](https://kubernetes.io/docs/concepts/overview/) cluster (for testing you can use [Docker Desktop](https://docs.docker.com/desktop/kubernetes/) or [minikube](https://minikube.sigs.k8s.io/docs/start)).

For quick start you can use provided [deploy.sh](https://github.com/golemcloud/golem/blob/main/kube/deploy.sh) script:

```bash copy
./deploy.sh -n golem
```

will deploy Golem with Redis, PostgreSQL and Nginx ingress to kubernetes namespace `golem`. Kubernetes [Persistent Volumes](https://kubernetes.io/docs/concepts/storage/persistent-volumes/) are used to store Golem's data.

For production is preferable to use separate PostgreSQL and Redis clusters.

See Golem Helm Chart [values.yaml](https://github.com/golemcloud/golem/blob/main/kube/golem-chart/values.yaml) and [readme](https://github.com/golemcloud/golem/blob/main/kube/README.md) for more details.
