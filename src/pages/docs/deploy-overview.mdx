# Deployment

Golem Services:

- [golem-component-service](https://github.com/golemcloud/golem/tree/main/golem-component-service) - [component](concepts-components) registry/management, see: [configuration](https://github.com/golemcloud/golem/blob/main/golem-component-service/config/component-service.toml), [environment variables](https://github.com/golemcloud/golem/blob/main/golem-component-service/config/component-service.sample.env)
- [golem-worker-service](https://github.com/golemcloud/golem/tree/main/golem-worker-service) - worker service providing APIs for workers and act like routing service for worker executors, see: [configuration](https://github.com/golemcloud/golem/blob/main/golem-worker-service/config/worker-service.toml), [environment variables](https://github.com/golemcloud/golem/blob/main/golem-worker-service/config/worker-service.sample.env)
- [golem-worker-executor](https://github.com/golemcloud/golem/tree/main/golem-worker-executor) - worker executor is responsible for running the [worker](concepts-workers) which belongs to assigned shards, see: [configuration](https://github.com/golemcloud/golem/blob/main/golem-worker-executor/config/worker-executor.toml), [environment variables](https://github.com/golemcloud/golem/blob/main/golem-worker-executor/config/worker-executor.sample.env)
- [golem-shard-manager](https://github.com/golemcloud/golem/tree/main/golem-shard-manager) - shard manager is a single node in charge of maintaining the worker executors shards assignments, only one instance can be alive at any given time, see: [configuration](https://github.com/golemcloud/golem/blob/main/golem-shard-manager/config/shard-manager.toml), [environment variables](https://github.com/golemcloud/golem/blob/main/golem-shard-manager/config/shard-manager.sample.env)
- [golem-component-compilation-service](https://github.com/golemcloud/golem/tree/main/golem-component-compilation-service) - sidecar service responsible for compiling [components](concepts-components), see: [configuration](https://github.com/golemcloud/golem/blob/main/golem-component-compilation-service/config/component-compilation-service.toml), [environment variables](https://github.com/golemcloud/golem/blob/main/golem-component-compilation-service/config/component-compilation-service.sample.env)

Golem Services are distributed as Docker images. They are available in [Golem Services Docker Hub](https://hub.docker.com/u/golemservices):

- [golem-component-service](https://hub.docker.com/r/golemservices/golem-component-service)
- [golem-worker-service](https://hub.docker.com/r/golemservices/golem-worker-service)
- [golem-worker-executor](https://hub.docker.com/r/golemservices/golem-worker-executor)
- [golem-shard-manager](https://hub.docker.com/r/golemservices/golem-shard-manager])
- [golem-component-compilation-service](https://hub.docker.com/r/golemservices/golem-component-compilation-service)
- [golem-router](https://hub.docker.com/r/golemservices/golem-router) - API router

Each Docker image is build for OS/ARCH-s:

- linux/amd64
- linux/arm64

Storage requirements:

- Redis
- PostgreSQL (or SQLite for testing purposes)
- File/Blob storage - Shared File System or S3

For deployment, see the following sections:

- [docker](#deploy-docker) deployment
- [kubernetes](#deploy-kubernetes) deployment
