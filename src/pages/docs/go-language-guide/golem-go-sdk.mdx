# Setting up the Golem Go SDK

<Callout type="warning">
  If the project was created with `golem-cli new`, it already has the Golem Go SDK set up and these
  steps can be ignored.
</Callout>

If setting up the Golem Go SDK manually, the following steps are required:

<Steps>
### Add the Golem Go SDK as a dependency

Add the following dependency to the project:

```shell
$ go get github.com/golemcloud/golem-go
```

### Add all the supported WIT files into the project

The project's `wit/deps` directory must contain all the WIT files from the [golem-wit](https://github.com/golemcloud/golem-wit/tree/main/wit/deps) repository.

### Create a second world in the WIT file importing all these WITs:

If the project's WIT file was like this:

```wit
package golem:demo;

world go-example {
  export api;
}
```

Modify it in the following way:

```wit
package golem:demo;

world go-example {
  export api;
}

world go-example-imports {
  import golem:api/host@0.2.0;
  import golem:rpc/types@0.1.0;

  import wasi:blobstore/blobstore;
  import wasi:blobstore/container;
  import wasi:cli/environment@0.2.0;
  import wasi:clocks/wall-clock@0.2.0;
  import wasi:clocks/monotonic-clock@0.2.0;
  import wasi:filesystem/preopens@0.2.0;
  import wasi:filesystem/types@0.2.0;
  import wasi:http/types@0.2.0;
  import wasi:http/outgoing-handler@0.2.0;
  import wasi:io/error@0.2.0;
  import wasi:io/poll@0.2.0;
  import wasi:io/streams@0.2.0;
  // import wasi:keyvalue/eventual-batch@0.1.0; // NOTE: cannot include it because the bindings collide with blobstore
  // import wasi:keyvalue/eventual@0.1.0; // NOTE: cannot include it because the bindings collide with blobstore
  import wasi:logging/logging;
  import wasi:random/random@0.2.0;
  import wasi:random/insecure@0.2.0;
  import wasi:random/insecure-seed@0.2.0;
  import wasi:sockets/ip-name-lookup@0.2.0;
  import wasi:sockets/instance-network@0.2.0;

  include go-example;
}
```

### Modify the build scripts

Make sure to pass `--world go-example` to the `wit-bindgen` call, so the generated bindings won't include the WASI and Golem APIs. Otherwise the build will fail due to duplicate bindings.

Pass `--world go-example-imports` to the `wasm-tools embed` command to import all the required interfaces in the final WebAssembly component.

</Steps>
